# Contributing

Thank you for your interest in contributing to the Guardrail API umbrella project. Please
review open issues, discuss substantial changes via GitHub discussions, and follow the
existing coding standards for Python, MkDocs, and deployment templates.

## Pull Requests
- Fork the repository and branch from `main`.
- Ensure tests, type checks, and documentation builds succeed locally before submitting.
- Provide clear descriptions of the problem solved, along with screenshots when applicable.

### IP & Branding
- Do not add company logos or trademarks to new assets without approval.
- Keep the “Guardrail Labs, LLC — Patent Pending” notice intact on docs landing and component pages.
